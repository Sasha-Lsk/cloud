# APK Studio (для AIDE, Android)

Простой декомпилятор / компилятор APK, собираемый в **AIDE** на Android без ПК.

- **APK → Smali** — полный разбор через apktool (ресурсы + код в .smali).
- **APK → Java** — читаемый Java-код через jadx (для понимания логики), плюс
  рядом создаётся собираемый smali-скелет `_buildable_smali`.
- **Smali → APK** и **Java → APK** — обратная сборка: apktool build (нативный
  aapt2) → zipalign → подпись apksig (v1+v2+v3).

## Как это работает без ПК

Движки `apktool.jar`, `jadx.jar`, `apksig.jar` уже собраны в формате Android
(`classes.dex` внутри), поэтому загружаются прямо в процесс приложения через
`DexClassLoader` и вызываются рефлексией — отдельная JVM/`java` не нужна.

Компиляция ресурсов при сборке требует нативного `aapt2`. Бинарники под все
ABI (`arm64-v8a`, `armeabi-v7a`, `x86`, `x86_64`) лежат в `assets/tools/bin/`;
нужный распаковывается в приватную папку и запускается apktool-ом.

## Сборка в AIDE

1. Открыть папку проекта `ApkStudio` в AIDE как Android-проект.
2. AIDE подхватит `AndroidManifest.xml`, `src/`, `res/`, `assets/`,
   `project.properties` (target=android-33).
3. Собрать и установить. `minSdkVersion=24`, `targetSdkVersion=33`.

> Ассеты (~60 МБ: движки + бинарники) — часть проекта, ничего докачивать не нужно.

## Рабочая папка (распаковка == сборка)

```
/storage/emulated/0/Download/ApkStudio/
  decompiler_apk/   ← результат декомпиляции (проекты smali/java)
  compiler_apk/     ← собранные .apk
```

## Раскладка файлов для успешной сборки (мировой стандарт apktool)

Проект после **APK → Smali** (`<имя>_smali/`):

```
AndroidManifest.xml   декодированный манифест
apktool.yml           метаданные (SDK, framework) — НЕ удалять!
res/                  ресурсы (layout, values, drawable, …)
smali/                код; multidex → smali_classes2/, smali_classes3/ …
assets/  lib/  original/  unknown/
```

Для сборки выбирайте именно папку `<имя>_smali` (с `apktool.yml`).
Для **Java → APK** выбирайте `<имя>_java` — внутри есть `_buildable_smali`,
он и собирается (правки Java-логики переносите в соответствующий smali).

## Чтобы пересобранное приложение работало на Android 8–12 без окна «разработано для более старой версии системы»

Это окно вызывает низкий `targetSdkVersion`. В распакованном проекте поднимите
его в `apktool.yml` (блок `sdkInfo` → `targetSdkVersion`) минимум до **30–33**.
`minSdkVersion` можно оставить низким (например 21) — APK установится и на
старые, и на новые Android, а предупреждение не появится. После правки —
пересоберите.

## Журнал и ошибки

Все шаги и ошибки (включая внутренний вывод apktool/jadx) пишутся в окно журнала
с отметками времени. Кнопка **⧉ Копир.** копирует весь журнал в буфер обмена —
по нему любой агент/человек поймёт причину сбоя. Кнопка **Очистить** очищает
журнал и сбрасывает выбранные в пикерах файлы/папки.
