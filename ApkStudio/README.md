# ApkStudio — декомпилятор / компилятор APK для Android (сборка в AIDE)

Простое приложение для телефона (без ПК), которое:

- **распаковывает APK → smali + xml** (движок apktool);
- **распаковывает APK → java** (движок jadx, для удобного чтения кода);
- **собирает smali/xml → APK**, выравнивает (zipalign) и **подписывает** (v1/JAR);
- **автоматически поднимает `targetSdkVersion`**, чтобы убрать системное окно
  «Это приложение разработано для более старой версии системы» на Android 8–12.

Всё работает прямо на устройстве через встроенные движки (dex-jar) и нативные
бинарники `aapt`/`aapt2`/`zipalign` под все ABI (arm64, armeabi-v7a, x86, x86_64).

---

## Рабочие папки

Папка распаковки = папка сборки:

```
/storage/emulated/0/Download/ApkStudio/
├── apks/         ← сюда кладёте исходный .apk
├── input_apk/    ← результат распаковки
│   ├── <имя>/        smali + ресурсы (для сборки обратно)
│   └── <имя>_java/   java-исходники (только для чтения)
├── output_apk/   ← готовый подписанный .apk
└── logs/         ← журналы
```

## Как собрать это приложение в AIDE

1. Установите **AIDE** из Play Market.
2. Скопируйте папку `ApkStudio` в память телефона
   (например в `/storage/emulated/0/AppProjects/ApkStudio`).
3. В AIDE: **Open** → выберите папку `ApkStudio`.
4. Дайте разрешения на файлы (AIDE попросит).
5. Нажмите **Run** (▶). AIDE соберёт и установит приложение.

> Проект — это стандартный Android Gradle-проект
> (`compileSdk 30`, `minSdk 21`, Java 8, androidx + Material).
> Движки лежат в `app/src/main/assets` и распаковываются при первом запуске
> в приватную папку приложения, где им ставится флаг «исполняемый».

### Зависимости (подтянет AIDE автоматически)
```
androidx.appcompat:appcompat:1.3.1
com.google.android.material:material:1.4.0
```

## Как пользоваться приложением

1. Положите APK в `Download/ApkStudio/apks/`.
2. Откройте ApkStudio, нажмите **Обновить**, выберите APK.
3. **В SMALI + XML** — для правки кода/ресурсов;
   **В JAVA** — чтобы прочитать логику (java нагляднее smali).
4. Отредактируйте нужные файлы в `input_apk/<имя>/`.
5. Выберите проект и нажмите **СОБРАТЬ APK**.
6. Установите файл из `output_apk/`.

Полная инструкция по раскладке файлов — в приложении, кнопка **Инструкция**.

## Про старые приложения (Android 8–12)

Окно «разработано для более старой версии системы» — это **не ошибка сборки**,
а системное предупреждение при слишком низком `targetSdkVersion`.
При сборке ApkStudio автоматически выставляет `targetSdkVersion = 30`
(в `apktool.yml` и манифесте), при этом **`minSdkVersion` не меняется**, —
приложение перестаёт показывать окно и нормально запускается на Android 8–12.

## Технические детали / ограничения (честно)

- Движки `apktool` и `jadx` включены как **dex-jar** (сконвертированы `d8`),
  поэтому запускаются на устройстве через `dalvikvm`.
- Подпись — **v1 (JAR), SHA-256 + RSA**. Проверено `apksigner verify`
  для API 21+ (Android 5–14). Для публикации в Play используйте свой ключ.
- `aapt2` обновлён до **34.0.4** (принимает ресурсы с `$` в имени —
  векторные анимации Material). Перед сборкой такие имена дополнительно
  нормализуются (`$` → `_`).
- Полный цикл (decode → build → sign) проверен на реальном APK.
- Очень свежие сложные APK (targetSdk 34/35, приватные ресурсы новых API)
  могут требовать ручной доработки ресурсов — это общее ограничение apktool,
  а не данного приложения. Обычные и старые приложения собираются штатно.

## Структура проекта

```
ApkStudio/
├── build.gradle, settings.gradle
└── app/
    ├── build.gradle
    └── src/main/
        ├── AndroidManifest.xml
        ├── java/com/apkstudio/
        │   ├── MainActivity.java   UI, запуск задач
        │   ├── GuideActivity.java  встроенная инструкция
        │   ├── Engine.java         ядро: распаковка/сборка/подпись
        │   ├── Patcher.java        фикс targetSdk и $-ресурсов
        │   ├── Signer.java         подпись v1 (JAR)
        │   ├── Pkcs7.java          DER/PKCS#7 подпись
        │   ├── Cfg.java, Log.java
        ├── res/                    тёмный Material UI
        └── assets/
            ├── engine/  apktool.jar, jadx.jar, testkey.pk8/.pem
            └── bin/<abi>/  aapt, aapt2, zipalign
```
